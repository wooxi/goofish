## 紧急修复：商品库页面打不开

### 现象
- 部分用户反馈点击「商品库」后页面无法打开（白屏/停留在当前页）。

### 根因
- 路由使用按需加载（动态 `import`）后，发布切换期间会出现旧页面缓存与新 chunk 文件不一致。
- 当浏览器尝试加载已失效的页面 chunk（典型是 `ProductLibraryPage-*.js`）时，会抛出 `Failed to fetch dynamically imported module`，导致目标页面无法进入。

### 修复方案（最小改动）
- 在前端路由层为懒加载页面增加 chunk 加载失败兜底：
  - 识别动态模块加载错误；
  - 首次命中时自动执行一次刷新重试；
  - 使用 `sessionStorage` 防止无限刷新循环；
  - 路由成功后清理重试标记。

## 风险与后续建议
1. **风险（低）**：若用户网络持续异常，自动刷新后仍可能失败（但不会循环刷新）。
2. **建议**：
   - 发布时保留历史静态资源一段时间，降低缓存错配窗口；
   - 后续接入全局前端错误监控（动态 import 失败率、路由失败率）；
   - 继续收敛大体积 chunk（当前仍有 >500kB warning），进一步降低首屏和切页抖动风险。
